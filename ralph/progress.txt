### Learnings for Future Iterations
- Vite 6 requires `sass` v1.70+ for the `sass.initAsyncCompiler` API — older sass versions fail silently or with cryptic errors
- The `mediatracker-api` local CJS package needs `build.commonjsOptions.include` and `optimizeDeps.include` in vite.config.ts for Rollup to resolve named exports
- Do NOT use `configFile: './babel.config.json'` in `@vitejs/plugin-react` — babel presets like `@babel/preset-env` convert ESM to CJS require() calls which break Vite's module resolution; only pass the specific plugins needed (e.g. `['macros']`)
- TypeScript 4.7.3 doesn't support `"moduleResolution": "bundler"` — use `"node"` instead
- Vite's default behavior copies the `public/` directory contents to the output root — font files end up both in `assets/` (hashed, referenced by CSS) and `fonts/` (raw copy from public/)
- `npm run build` in package.json no longer needs `cross-env NODE_ENV=production` — Vite sets it automatically for `vite build`
