The native implementation was a workaround for React 17 Radix compatibility issues that are now resolved with React 19 (upgraded in US-008). The new implementation maintains the same public API, auto-expand behavior on `hasActiveSelection`, and keyboard accessibility (Enter/Space toggle) via Radix. Removed `@radix-ui/react-collapsible` from package.json (it was listed as a direct dependency but never directly imported — shadcn/ui Collapsible brings it transitively via the unified `radix-ui` package). Updated FacetSection.test.tsx to verify aria-controls linking without asserting on the specific role="region" implementation (Radix handles this).

- **Files changed**:
  - `client/src/components/ui/collapsible.tsx` — NEW: shadcn/ui Collapsible component (installed via `npx shadcn@latest add collapsible`); added React import to fix JSX reference issue
  - `client/src/components/Facets/FacetSection.tsx` — Complete rewrite using Collapsible, CollapsibleTrigger, CollapsibleContent; removed manual ID generation and event handlers; applied zinc design tokens to replace gray/slate colors
  - `client/src/components/__tests__/FacetSection.test.tsx` — Updated single test assertion for aria-controls linking (Radix implementation detail); all 8 tests passing
  - `client/package.json` — Removed `@radix-ui/react-collapsible` (now transitive via radix-ui unified package)

- **Verification**:
  - Typecheck: 0 errors
  - Tests: 74 suites, 1173 tests, all passing (8 FacetSection-specific tests passing)
  - Build: Production build completes successfully; bundle size unchanged (no new dependencies added, old one removed)
  - Browser: Dev server loads without Collapsible-related JS errors (pre-existing API 500 errors from backend are unrelated)

- **Learnings for future iterations**:
  - shadcn/ui Collapsible (Radix) does NOT need mocking in jest tests — it works correctly in jsdom without custom mocks (unlike Radix Dialog/Sheet which need context-based state sharing)
  - Radix Collapsible automatically manages aria-expanded and aria-controls on the trigger element — no need to manually generate IDs or ARIA attributes
  - The native HTML workaround in FacetSection.tsx (using `useRef` to generate unique IDs) is no longer needed with React 19 + Radix Collapsible
  - `@radix-ui/react-collapsible` as a direct dep is redundant when using shadcn/ui Collapsible — the unified `radix-ui` package provides it
  - Color token migration: `text-gray-800 dark:text-slate-200` → `text-zinc-800 dark:text-zinc-200`; `hover:bg-gray-100 dark:hover:bg-slate-700` → `hover:bg-zinc-100 dark:hover:bg-zinc-700`; these align with shadcn/ui zinc palette
