## Codebase Patterns
- TanStack React Query v5 object-form API throughout
- `Parameters<typeof apiFn>[0]` used for typed mutationFn wrappers
- Global `select` function on QueryClient for error envelope detection
- React 19 uses `createRoot` from `react-dom/client` — no more `ReactDOM.render`
- React 19 `FunctionComponent<P>` no longer implicitly includes `children` — must declare `children: React.ReactNode` explicitly
- React 19 `useRef<T>()` requires exactly 1 argument — use `useRef<T>(null)`
- React 19 `JSX` namespace must be qualified: `React.JSX.Element` (not bare `JSX.Element`)
- `@testing-library/react` v16 exports `renderHook` and `act` — no more custom polyfills
- `@lingui/react/macro` declares `Trans` returning `ReactNode` but TS 4.7.3 needs `ReactElement | null` for JSX — keep lingui-macro.d.ts type override until TS 5.1+ upgrade
- `@react-spring/web` v9.4.4 and `styled-components` v5.3.5 types are incompatible with `@types/react` v19 — use `@ts-expect-error` suppressions
- `--legacy-peer-deps` still needed for npm install due to some transitive deps not listing React 19 in peerDependencies
- FullCalendar v6 moves type exports to `@fullcalendar/core` instead of `@fullcalendar/react`
- Vite 6 build: use `@vitejs/plugin-react` with `babel: { plugins: ['@lingui/babel-plugin-lingui-macro'] }` for Lingui v5
- `mediatracker-api` is a local CJS package (`file:../rest-api`) — needs `build.commonjsOptions.include` and `optimizeDeps.include` in vite.config.ts
- tsconfig.json uses `"moduleResolution": "node"` (TS 4.7.3 doesn't support `"bundler"`)
- Vite `resolve.alias`: `{ src: path.resolve(__dirname, './src') }` maps `src/` imports
- Font woff2 files are served as hashed assets (not inlined) under Vite — this is fine and better for caching
- `sass` package must be v1.70+ for Vite 6 `sass.initAsyncCompiler` API
- Tailwind v4 CSS-first config: `@import "tailwindcss"` in `tailwind.css`, no `tailwind.config.js`
- Global base/reset styles MUST be in `@layer base` inside `tailwind.css` — unlayered CSS beats layered Tailwind utilities
- Standalone CSS files using `@apply` with Tailwind classes need `@reference "tailwindcss"` at the top
- `@tailwindcss/vite` needs a manual `.d.ts` type declaration (TS 4.7.3 + `moduleResolution: "node"` can't resolve `.d.mts`)
- `dark.css` content is now in `tailwind.css @layer base` — do NOT re-import dark.css
- shadcn/ui CSS variables (`--primary`, `--background`, etc.) must go in `:root` inside `@layer base`, NOT in `@theme` — `@theme` is for Tailwind design tokens
- Tailwind v4 `@theme` registers `--color-*` tokens that generate `bg-*`/`text-*`/`border-*` utilities — use `--color-primary: var(--primary)` in `@theme` to bridge shadcn CSS vars → Tailwind utilities
- Jest `moduleNameMapper` needs `'^@/(.*)$': '<rootDir>/src/$1'` entry to resolve `@/` alias in test files
- shadcn/ui Button uses `@radix-ui/react-slot` for the `asChild` prop — install it alongside cva and tailwind-merge
- clsx v2+ uses a default export: `import clsx from 'clsx'` not `import { clsx } from 'clsx'`

### Learnings for Future Iterations
- React 19 `FunctionComponent<P>` removes implicit `children` — every component accepting children must declare `children: React.ReactNode` in its props
- `useRef<T>()` signature changed in @types/react v19 — requires exactly 1 argument (`useRef<T>(null)`)
- The global `JSX` namespace is no longer auto-exported — use `React.JSX.Element` instead
- `@testing-library/react` v16 natively exports `renderHook` and `act` — replace all custom polyfills
- TypeScript 4.7.3 doesn't support the new JSX element type narrowing in @types/react v19 — components returning `ReactNode` (like Lingui's `Trans`) need type overrides via tsconfig paths until TS 5.1+
- `@react-spring/web` v9.4.4 and `styled-components` v5.3.5 types are incompatible with @types/react v19 — requires `@ts-expect-error` suppressions (12 total across 5 files)
- `createRoot` returns a root object with `.render()` and `.unmount()` — for imperative rendering (like Confirm.tsx), call `root.unmount()` before removing the DOM node
