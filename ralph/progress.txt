## 2026-03-06 - US-005
- Created server/src/services/recommendations/IgdbSimilarClient.ts
- Created server/__tests__/services/recommendations/IgdbSimilarClient.test.ts
- IgdbSimilarClient implements the two-step IGDB query pattern:
  - Step 1: POST /v4/games with `fields similar_games; where id = {igdbId};` to get similar game IDs
  - Step 2: POST /v4/games with `fields name,total_rating,total_rating_count; where id = (id1,id2,...);` to batch-fetch details
- Constructor accepts IgdbSimilarClientDeps interface: { requestQueue, clientId, clientSecret, axiosInstance? }
  - RequestQueue is injected (shared with IGDB metadata provider) to respect 4 req/sec rate limit
  - clientId and clientSecret injected for OAuth2 token management
  - Optional axiosInstance for testing
- OAuth2 token management with 60-second expiry buffer:
  - expiresAt = tokenAcquiredAt.getTime() + token.expires_in * 1000
  - needsRefresh = expiresAt - Date.now() < 60_000
  - Token refresh routed through shared RequestQueue
- Normalizes total_rating (0-100) to externalRating (0-10) by dividing by 10
- total_rating of 0 or undefined → externalRating = null
- Validates externalRating is null or within [0.0, 10.0]
- All API requests (token + Step 1 + Step 2) flow through the injected RequestQueue
- 28 unit tests covering all acceptance criteria:
  - Two-step flow (4 tests), total_rating normalization (7 tests), SimilarItem mapping (3 tests)
  - Token management (3 tests), Shared RequestQueue usage (2 tests), Error propagation (3 tests)
  - Edge cases (4 tests), API request format (2 tests)
- TypeScript type check passes with no errors
- All 73 recommendation + entity tests pass (no regressions)
- Commit: feat: US-005 - Implement IgdbSimilarClient for games (two-step query with shared RequestQueue)

**Learnings for future iterations:**
- The IGDB provider's RequestQueue, token, and tokenAcquiredAt are all `private` — cannot be accessed directly from outside the class
- Best DI pattern for shared queue: accept RequestQueue instance via constructor deps interface, along with clientId/clientSecret for independent OAuth management
- IGDB uses POST requests with Apicalypse query strings as body (not JSON) — e.g., `fields similar_games; where id = 1000;`
- The Game interface in igdb.ts has `similar_games?: number[]` — these are IDs only, requiring a second query for details
- IGDB batch query syntax: `where id = (101,102,103);` to fetch multiple games in one call
- Token expiry check pattern: `expiresAt - Date.now() < 60_000` provides 60-second buffer before actual expiry
- The IGDB `total_rating` field uses 0-100 scale while SimilarItem expects 0-10 — always divide by 10
- Mock RequestQueue pattern: `{ request: jest.fn(async (fn) => fn()) }` executes immediately without delay
