## Codebase Patterns
- Logger import: `import { logger } from 'src/logger'` (not 'src/logger/logger')
- Module path aliases: `src/` maps to `server/src/` (baseUrl is `.` in tsconfig with `src/` prefix)
- Config class uses static readonly fields for raw env values
- jest.mock() factory cannot reference out-of-scope variables (use jest.fn() directly inside factory, then import the mock and use it)
- Tests use `jest.mock('src/...')` for module mocking with babel-jest
- tmdbLang array and TmdbLang type are in `server/src/entity/configuration.ts`
- Config static fields are `static readonly` — cast to `unknown as { field: type }` in tests to mutate mock
- TV show `localizedDetails()` mirrors `details()` structure: 1 show call + N season calls with per-season episode extraction
- Season/episode ID mapping in updateMediaItem: build Maps from `updatedMediaItem.seasons` (which has DB IDs after `margeTvShow()`) keyed by `seasonNumber` and `seasonNumber:episodeNumber`
- `updatedMediaItem.seasons` type is a union — cast to `TvSeason[]` when inside `mediaType === 'tv'` check
- Season-level API failures in localizedDetails: catch per-season, set episodes to `[]`, log error, continue with remaining seasons
- `upsertSeasonTranslation` and `upsertEpisodeTranslation` follow exact same pattern as `upsertMediaItemTranslation` (async functions, not class-based)
- Translation loop in updateMediaItem: first-language data from base `details()` + localized data for each language from `localizedDetails()` — both movie-level AND season/episode level for TV
