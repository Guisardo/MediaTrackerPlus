- WatchlistWriter.write(userId, items, estimatedRating) processes SimilarItem[] into the user's watchlist
- For each SimilarItem:
  1. Calls findMediaItemByExternalId to resolve external ID → mediaItem row

  3. Applies one of: INSERT (new), UPDATE (minimum-wins), or SKIP
- Media type mapping: 'game' → 'video_game', others pass through
- External ID mapping: movie/tv → { tmdbId: Number(id) }, game → { igdbId: Number(id) }, book → { openlibraryId: id }
- Minimum-wins strategy: update estimatedRating only when incoming value is strictly lower
- Items with null estimatedRating get the incoming value (treated as update)
- Watched = media-item-level seen row (episodeId IS NULL); episode-level seen does not block
- Rated = media-item-level userRating row (seasonId=null, episodeId=null); season/episode ratings don't block
- Import failure (findMediaItemByExternalId returns undefined): log WARN, skip, continue batch
- Returns WriteResult { added, updated, skipped } for RecommendationService observability
- Exports WatchlistWriterDeps interface for constructor injection (testability)
