# Documenter

Use this skill for documentation work across the repository.

## Use This Skill For

- README cleanup and refreshes
- Markdown normalization
- Docs audits and structure reviews
- Documentation reorganization
- Index, TOC, or internal-link maintenance

## Workflow

1. Scan the current documentation scope before editing broadly.
2. Normalize formatting and structure without changing meaning unless the task explicitly asks for content edits.
3. Follow project-specific documentation conventions before moving files or rebuilding indexes.
4. Treat generated documentation artifacts as generated; do not hand-edit them.
5. After structural docs changes, verify links, indexes, and navigation artifacts still match the repository state.
