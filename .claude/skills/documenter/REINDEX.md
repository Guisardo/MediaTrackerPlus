# Reindexing Reference

Reindexing rebuilds navigation artifacts: the master index, per-directory tables of contents, anchor references, and cross-link validity. Run after normalization and organization are complete.

---

## What Reindexing Covers

| Artifact | File | Description |
|----------|------|-------------|
| Master index | `docs/INDEX.md` | Single entry point listing every doc file |
| Directory READMEs | `docs/*/README.md` | Per-section file lists with one-liner descriptions |
| Table of contents | Inside each large `.md` file | Auto-generated `## Contents` section |
| Cross-references | All relative links in `.md` files | Verified to resolve after file moves |
| API index | `docs/api/README.md` | Endpoints extracted from `server/openapi.json` |

---

## Reindexing Rules

### R1 — Master Index (`docs/INDEX.md`)

The master index must exist and be the single navigation hub.

**Structure:**

```markdown
# Documentation Index

> Last updated: YYYY-MM-DD — generated by `build_index.py`

## Quick Links

- [Getting Started](../README.md)
- [API Reference](api/README.md)
- [Configuration](configuration/README.md)
- [Deployment](deployment/README.md)
- [Integrations](integrations/README.md)
- [Development](development/README.md)

## All Files

### Root
| File | Description |
|------|-------------|
| [README.md](../README.md) | Project overview and quick-start |
| [CONTRIBUTING.md](../CONTRIBUTING.md) | Contribution guidelines |
| [CHANGELOG.md](../CHANGELOG.md) | Version history |

### API
| File | Description |
|------|-------------|
| [api/README.md](api/README.md) | API overview |

... (continued per directory)
```

**Rules:**
- Never hand-edit this file — always regenerate with `build_index.py`
- Add `> Last updated:` timestamp on every regeneration
- Sort entries alphabetically within each section
- Each row: filename link + one-sentence description (extracted from file's H1 or frontmatter `title`)

### R2 — In-file Table of Contents

For any `.md` file with more than **5 H2 sections**, insert a `## Contents` section immediately after the H1:

```markdown
# Document Title

## Contents

- [Section One](#section-one)
- [Section Two](#section-two)
  - [Subsection](#subsection)
- [Section Three](#section-three)

---

## Section One
...
```

**Anchor rules:**
- GitHub-flavored Markdown anchors: lowercase, spaces → hyphens, strip special chars except `-`
- `## Section Two — Details` → `#section-two--details`
- `### Rule N1 — File Encoding` → `#rule-n1--file-encoding`

### R3 — Cross-Reference Validation

Every relative link in every `.md` file must resolve. Broken links are hard errors.

Categories:
1. **File links** — `[text](path/to/file.md)` — target file must exist
2. **Anchor links** — `[text](file.md#anchor)` — anchor must exist in target file
3. **Image links** — `![alt](path/to/img.png)` — image file must exist

The `check_links.py` script validates all three categories and outputs a structured report.

### R4 — API Index from OpenAPI Spec

`docs/api/README.md` must be regenerated from `server/openapi.json` whenever the spec changes.

**Format:**

```markdown
# API Reference

> Source: `server/openapi.json` — last regenerated YYYY-MM-DD

Full interactive documentation: https://dnlwttnbrg.github.io/MediaTrackerPlus/

## Endpoints

### Authentication
| Method | Path | Summary |
|--------|------|---------|
| POST | `/api/v1/auth/login` | Authenticate user |

### Media Items
| Method | Path | Summary |
|--------|------|---------|
| GET | `/api/v1/items` | List all media items |
...
```

The `build_index.py --api` flag handles this extraction.

## Running Reindex Scripts

```bash
# Full reindex: rebuild INDEX.md, all directory READMEs, in-file TOCs
python .claude/skills/documenter/scripts/build_index.py

# Rebuild only the master index
python .claude/skills/documenter/scripts/build_index.py --index-only

# Rebuild only the API index from openapi.json
python .claude/skills/documenter/scripts/build_index.py --api

# Check all internal links (dry run — no modifications)
python .claude/skills/documenter/scripts/check_links.py

# Check links and output JSON report
python .claude/skills/documenter/scripts/check_links.py --json > link-report.json

# Fix anchors in a specific file (rewrites TOC to match current headings)
python .claude/skills/documenter/scripts/build_index.py --toc README.md
```

---

## Reindex Checklist

After running the scripts, verify:

- [ ] `docs/INDEX.md` exists and has a `Last updated` timestamp
- [ ] Every `docs/*/README.md` lists all files in its directory
- [ ] Files with 5+ sections have a `## Contents` TOC
- [ ] `check_links.py` exits with code 0 (no broken links)
- [ ] `docs/api/README.md` reflects the current `server/openapi.json`
- [ ] Root `README.md` links to `docs/INDEX.md` for extended reading

---

## Manual Override: When Not to Regenerate

Do **not** regenerate `docs/INDEX.md` if:
- The directory structure changed but files were not yet committed (links would break)
- `scan_docs.py` has not been run since the last file move

Always run in this order: `scan_docs.py` → `normalize_markdown.py` → `build_index.py` → `check_links.py`.
